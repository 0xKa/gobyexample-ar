# لغة Go بالأمثلة

ترجمة عربية مستقلة لموقع [Go by Example](https://gobyexample.com)، وهو مدخل عملي إلى لغة Go يعتمد على برامج نموذجية مشروحة.

> Arabic translation of [Go by Example](https://github.com/mmcgrana/gobyexample). This translation is maintained as a separate website and follows the upstream examples and build toolchain.

الموقع المخطط له بعد اكتمال الترجمة: `https://0xKa.github.io/gobyexample-ar/`

## حالة المشروع

- عناوين الأمثلة العربية: مكتملة، مع إبقاء روابط الأمثلة الإنجليزية ثابتة.
- واجهة الموقع واتجاه الكتابة من اليمين إلى اليسار: مكتملان.
- ترجمة شروح الأمثلة: قيد التنفيذ؛ راجع [جدول التقدم](PROGRESS.md).
- النشر عبر GitHub Pages: سيُفعّل بعد اكتمال الترجمة والمراجعة.

لا يُعد الموقع جاهزًا للنشر العام حتى تكتمل شروح الأمثلة الحالية وتُراجع لغويًا وتقنيًا.

## كيفية عمل المشروع

توجد النصوص الأصلية والشيفرة في `examples`. تستخرج أداة البناء الشروح من ملفات `.go` و`.sh`، ثم تدمجها مع القوالب الموجودة في `templates` وتولد موقعًا ثابتًا داخل `public`.

| المسار | الغرض |
| --- | --- |
| `examples/` | الشيفرة والشروح التي تمثل مصدر المحتوى الأساسي |
| `examples.txt` | ترتيب الأمثلة وأسماؤها الأصلية المستخدمة لاشتقاق الروابط الثابتة |
| `examples.ar.txt` | العناوين العربية المرتبطة بمعرّفات الأمثلة الثابتة |
| `templates/` | قوالب HTML وملفات CSS وJavaScript والأصول المشتركة |
| `tools/` | أدوات الاختبار والبناء والتوليد والخدمة المحلية |
| `public/` | الموقع الثابت المولّد؛ لا يُحرر يدويًا |

## البناء والمعاينة محليًا

تحتاج إلى إصدار Go المحدد في [`go.mod`](go.mod) وبيئة Bash. على Windows يمكن استخدام Git Bash أو WSL.

لبناء الموقع وتحديث `public`:

```console
$ tools/build
```

للتحقق من أن الملفات المولّدة مطابقة للمصادر:

```console
$ VERBOSE=1 TESTING=1 tools/build
```

لمعاينة الموقع:

```console
$ tools/serve
```

ثم افتح `http://127.0.0.1:8000/`.

## المزامنة مع المشروع الأصلي

هذا المستودع ترجمة مستقلة وليس بديلًا عن المشروع الأصلي. تُجلب إصلاحات الشيفرة والأمثلة الجديدة من [`mmcgrana/gobyexample`](https://github.com/mmcgrana/gobyexample)، ثم تُترجم الشروح الجديدة قبل دمجها في فرع النشر.

يجب اقتراح الأمثلة الجديدة وتغييرات بنية التنقل على المشروع الأصلي أولًا، لأن هذه الترجمة تهدف إلى البقاء متوافقة معه.
